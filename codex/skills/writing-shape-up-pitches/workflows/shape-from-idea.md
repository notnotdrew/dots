# Shape From Idea Workflow

Use this when the user brings a rough idea or problem statement.

1. Extract the core problem.
   - Who feels it?
   - What breaks today?
   - Why is it worth a bet now?

2. Get the appetite.
   - If it is missing, ask for it before finalizing the pitch.

3. Shape the solution.
   - Name the product area and main elements.
   - Keep the description rough enough to avoid premature implementation detail.

4. Surface rabbit holes and no-gos.
   - Name the unknowns that could blow the appetite.
   - State the boundaries that should survive into execution.

5. Stress-test the draft.
   - Check that the appetite fits the shaped solution.
   - Check that the solution actually addresses the problem.

6. Return the pitch using [../references/template.md](../references/template.md).
