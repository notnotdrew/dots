# Refine Draft Workflow

Use this when the user already has a draft pitch.

1. Read the draft completely.
2. Audit it for:
   - weak or generic problem framing
   - missing appetite
   - over-detailed solution language
   - hidden rabbit holes
   - vague or missing no-gos
3. Keep the existing useful shape, but tighten the weak sections.
4. Rewrite the draft into the canonical structure when needed.
5. Return the improved pitch plus any critical missing inputs that still block a credible bet.
