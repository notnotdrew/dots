# Distill Notes Workflow

Use this when the user provides research, meeting notes, or scattered source material.

1. Extract the signals:
   - repeated pain points
   - evidence and concrete examples
   - affected users or teams
   - constraints already known

2. Collapse the material into one bounded problem.
   - If the notes contain multiple unrelated bets, split them instead of forcing one pitch.

3. Decide the appetite.
   - Ask only if the source material does not already imply it.

4. Shape the solution and boundaries.
   - Name the main product area
   - Name the key elements
   - Name rabbit holes and no-gos

5. Return the final pitch using [../references/template.md](../references/template.md).
