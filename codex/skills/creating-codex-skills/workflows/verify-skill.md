# Verify Skill Workflow

Use this workflow before considering a skill complete.

## Checklist

- [ ] Frontmatter is valid and matches the directory name
- [ ] `description` states both the job and the trigger conditions
- [ ] `Quick Start` reaches the core behavior quickly
- [ ] Local links resolve to files that exist
- [ ] `SKILL.md` stays focused and pushes detail into supporting files when needed
- [ ] Examples are concrete and match the intended output
- [ ] No copied assistant-specific terminology remains unless intentionally adapted

## Final Pass

Read the skill like a fresh caller:

- Can you tell when to invoke it from the first screen?
- Is the routing obvious?
- Are the constraints durable instead of temporary?
- Would a user need outside context to use it correctly?
