# Audit Skill Workflow

Use this workflow when the user wants feedback on an existing skill.

1. Read `SKILL.md` completely.
2. Identify the actual job the skill claims to do.
3. Audit against these questions:
   - Does the description clearly state what the skill does and when to use it?
   - Does `Quick Start` get to value quickly?
   - Is `SKILL.md` carrying too much detail that should move to supporting files?
   - Are examples concrete enough to teach the behavior?
   - Are there stale tool assumptions or broken references?
   - Is the skill too broad, too vague, or too mode-heavy for its current structure?
4. Report the smallest set of changes that materially improves the skill.
5. If the user wants edits, make them without widening into unrelated refactors.
