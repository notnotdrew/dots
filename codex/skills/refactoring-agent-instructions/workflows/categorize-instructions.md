# Phase 3: Categorize Instructions

Group non-root rules into a small set of files.

Use [references/category-templates.md](../references/category-templates.md) when a category needs structure hints.

## Procedure

1. Let categories emerge from the content.
2. Keep 3 to 8 categories when the material is substantial.
3. Merge tiny categories. Split only when a file becomes hard to scan.
4. Assign each instruction to one best file. Do not duplicate rules.
5. Order each file by impact, frequency, then theme.

## Common Destinations

- `testing.md`
- `git-workflow.md`
- `architecture.md`
- `code-style.md`
- `typescript.md`
- `security.md`
- `documentation.md`

## Output

- Final category map
- Rough line counts
- User-confirmed structure when grouping is materially ambiguous
