# Phase 4: Create Structure

Write the new root and `guidelines/` files.

Use [references/category-templates.md](../references/category-templates.md) only when a file needs a clearer shape.

## Procedure

1. Create `guidelines/` at the project root.
2. Write one file per category:
   - direct title matching the filename
   - self-contained rules
   - short bullet lists over prose
3. Rewrite the root file to include only:
   - project identity
   - non-obvious commands
   - critical overrides
   - links to `guidelines/*.md`
4. If multiple source roots exist, choose one primary root and keep the others until deletion is approved.
5. Verify every relative link and re-check root length.

## Output

- Rewritten root file
- `guidelines/` directory
- Link check and root line count
